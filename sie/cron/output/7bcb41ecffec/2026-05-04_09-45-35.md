cloudflared is at `~/.local/bin/cloudflared` on the host.

### One-time tunnel creation

```bash
ssh -o StrictHostKeyChecking=no sean@localhost "~/.local/bin/cloudflared tunnel create agent-os 2>&1"
```

This outputs:
- Tunnel UUID (used for `tunnel run --token`)
- Credentials file at `~/.cloudflared/<uuid>.json`

### Route tunnel to DNS

```bash
ssh -o StrictHostKeyChecking=no sean@localhost "~/.local/bin/cloudflared tunnel route dns agent-os-tunnel appexample.codeovertcp.com"
```

### Get tunnel token

The tunnel token is used in `CLOUDFLARED_TUNNEL_TOKEN` env var. Get it from:
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "cat ~/.cloudflared/<tunnel-uuid>.json | python3 -c 'import json,sys; d=json.load(sys.stdin); print(d[\"token\"])'"
```

### Run tunnel manually (for testing)


## agent-os: Cloudflare Tunnel Setup

**Goal**: Get the Cloudflare tunnel running so agent-os is accessible at os.codeovertcp.com.

**Load the `infrastructure-as-code` skill first**, then follow these steps.

### Context
- cloudflared is installed at `~/.local/bin/cloudflared` on the host
- Terraform files are at `/home/sean/.hermes/agent-os/infra/terraform/`
- docker-compose.yml expects `CLOUDFLARED_TUNNEL_TOKEN` in `.env`

### Steps

**Step 1 — Check if tunnel already exists**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "ls ~/.cloudflared/ 2>/dev/null && echo 'cloudflared dir exists' || echo 'no cloudflared dir'"
```

**Step 2 — Create Cloudflare tunnel (if not exists)**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "~/.local/bin/cloudflared tunnel create agent-os 2>&1"
```

**Step 3 — Route tunnel to DNS**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "~/.local/bin/cloudflared tunnel route dns agent-os os.codeovertcp.com 2>&1"
```

**Step 4 — Get tunnel token**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "ls ~/.cloudflared/ && cat ~/.cloudflared/*.json | python3 -c 'import json,sys; d=json.load(sys.stdin); print(\"Token:\", d.get(\"token\",\"NOT_FOUND\"))' 2>&1"
```

**Step 5 — Update .env with tunnel token**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "sed -i 's|PASTE_YOUR_TUNNEL_TOKEN_HERE|<token_from_step4>|' /home/sean/.hermes/agent-os/.env 2>&1"
```
Replace `<token_from_step4>` with the actual token from step 4.

**Step 6 — Start cloudflared manually (test)**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "cd /home/sean/.hermes/agent-os && CLOUDFLARED_TUNNEL_TOKEN=<token> ~/.local/bin/cloudflared tunnel run --token <token> agent-os 2>&1 &
sleep 5
docker ps | grep cloudflared"
```

**Step 7 — Check tunnel URL**
```bash
ssh -o StrictHostKeyChecking=no sean@localhost "~/.local/bin/cloudflared tunnel info agent-os 2>&1"
```
