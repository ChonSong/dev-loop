## Daily Workflow (30 min)

1. Pick 3 categories from today's rotation
2. Open SEEK, paste 3 search queries
3. Scan results, pick 5 matching roles
4. Open matching CV PDF from Drive folder
5. Submit application (tweak 1-2 lines for the specific role)
6. Log in tracker: Date, Company, Role, Category, Platform, Salary, Link, "applied", CV version

## Weekly Target

- 25+ applications per week
- At least 6 of 9 categories hit
- Sunday review: update statuses, follow up on applications >7 days old, adjust queries if a category yields no responses in 2 weeks

## Search Query Template

Each category needs 4-5 SEEK queries. Pattern:
```
junior/entry-level/graduate [ROLE KEYWORDS] Sydney
```

Examples for Python-Backend:
```
junior python developer Sydney
junior backend engineer Sydney
associate backend developer Sydney
junior software engineer python Sydney
entry level python developer Sydney
```

## CV Variation Strategy

Each CV shares the same visual identity (accent bar, contact layout, font) but differs in:

| Element | How It Changes Per Category |
|---------|-----------------------------|
| Subtitle | Matches the target role title (e.g., "Data Engineer" vs "Junior ML Engineer") |
| Skills | Only show the most relevant skill groups for that role |
| Summary | First 2 sentences lead with category-appropriate experience |
| Experience bullets | Reorder so the most relevant bullets are at the top; drop unrelated ones |
| Projects | Reorder so category-relevant projects appear first |
| Philosophy | Tweak the principle to match the role context (simplicity in modeling vs simplicity in architecture vs simplicity in analysis) |
