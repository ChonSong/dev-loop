# Sean's CV Preferences

## Formatting
- Body text: 8.5pt minimum, line-height 1.5-1.55
- Job titles: 9pt bold
- Bullets: 8pt, line-height 1.5
- Skills section: 8.5pt, two-column layout
- Contact: 8.5pt
- Headings (h2): 10pt bold
- Philosophy box: #f8fafc bg, #2d7d6f left border, 8pt

## Page
- Exactly 1 page (A4, 210mm × 297mm = ~1123px tall)
- Blank space target: <5%
- Footer pinned to bottom via flexbox margin-top:auto
- No overflow to page 2

## Content
- Lead with infrastructure/automation projects for systems roles
- Include Engineering Philosophy (Principle of Parsimony) as a differentiator
- Every bullet must be evidence-backed from actual source code (library names, metrics, architecture patterns)
- Cut data science / ML projects unless directly relevant to the target role
- The autonomous development pipeline (Coach/Player loop) is a unique signal worth including

## Excluded Projects (DO NOT include in CVs)
- casaos-agent
- hermes-sync

## Cover Email Style
- Direct, no filler ("Hi," not "Dear Hiring Team" or "I hope this finds you well")
- 5-6 short paragraphs max
- Lead with the self-hosted infrastructure story
- Mention specific technologies and numbers
- Close with GitHub link
