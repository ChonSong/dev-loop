
The engine correctly detects and reports clone failures, but Phase 2 still proceeded with tasks that had no chance of succeeding. The task preconditions (repo presence) were never checked before selecting tasks for execution.

### Report bug: "What We Completed" lists old completed tasks

The Phase 3 reporter incorrectly picked up `status: done` tasks from the roadmap as if they were executed this session:
- task-openclaw-006 through task-openclaw-014 all listed as "Result: TDD complete..." with "Tests: 0p/0f"
- These tasks were marked `done` on 2026-05-01 — they were not touched this session

The bug is in `_record_task_result` or the report generator — tasks with `status: done` are being included in the "completed this session" section.

## Learnings

1. **Read-only filesystem silently kills Phase 2.** Clone fails → fetch fails → push fails → nothing to execute against.

4. **TODO scanner findings heavily skewed.** 183 findings, all from hermes-agent SKILL.md/README.md — deduplicate by `{project, file, lineno}`.
