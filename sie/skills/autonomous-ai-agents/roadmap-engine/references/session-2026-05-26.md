## Root Causes Found

### 1. Clone `cwd` was wrong (patched mid-session)
`roadmap_engine.py` line ~131 used `cwd=HERMES_SYNC / "workspace" / "projects"` for git clone.
Should be `cwd=WORKSPACE_TMP / "projects"`. Fixed with a 1-line patch.

Affects: All Phase 1 clone/fetch operations that use a `cwd` parameter.

### 2. `repo-transmute` is an empty repo
Clone succeeded but contains only an empty `tests/` directory.
No `translator.py`, no `pyproject.toml`, no source code. This is why
`task-001` fails — the target file literally doesn't exist.

### 3. `everything-dashboard` has no test framework
TypeScript/React project has no `package.json` test script, no `jest.config.ts`,
no `vitest.config.ts`. `_execute_test()` correctly identifies the gap but
the error message is generic "No test framework detected" with no diagnostics
about what was checked.

### 4. `_execute_test()` returns wrong status on block
```python
# Current (wrong):
return {"status": "done", "result": "No test framework detected", ...}

# Should be:
return {"status": "blocked", "result": "No test framework detected", ...}
```
Because `_record_task_result()` only calls `roadmap.complete_task()` for
`status=="done"`, blocked tasks stay in `todo` and get re-selected every night.

### 5. Stale task results polluting reports
The 9 `task-openclaw-*` tasks in `roadmap.json` all have `result="TDD complete: ..."`
with `commits=0` and `tests_passed=0` set from previous sessions where no code
was actually written. These get echoed verbatim in "What We Completed" every
time the report generates. Should be cleared or set to `status=blocked`.
