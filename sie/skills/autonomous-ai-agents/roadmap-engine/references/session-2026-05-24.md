The lint gate runs `ruff check .` inside the project directory. If the directory doesn't exist, it raises a non-Retryable error and reverts the patch. The patch itself may have been fine — it was never tested.

## Root Causes

1. **openclaw repo never cloned** — `ChonSong/openclaw` was set as `proj-005` but the clone step in Phase 1 failed silently (or was never attempted)
2. **Lint gate checks presence before applying** — the executor checks `local.exists()` before running the coder, but the lint gate also runs inside the project directory and fails if the path doesn't exist
3. **No pre-flight check** — there's no task-level guard that marks openclaw tasks as `blocked` when the repo is missing (unlike how `proj-005` was set to `'blocked'` in the May 18 implementation notes)
