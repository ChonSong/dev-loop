# Session 2026-06-18 — Dev Loop Stalled, Coach Cron Erroring

## Problem
The GTO Wizard study page was missing basic features (action buttons for Fold/Call/Raise/Check/Bet/All-in). The user asked "why didn't the system figure out these features were missing?"

## Root Cause Chain
1. **Coach cron erroring** — `coach-development-loop` had `last_status: error` for multiple runs. The Phase 2.5 spec gap detection (which would have compared `docs/reference-study-interface.png` against the live `/study` page and flagged missing buttons) never ran.
2. **`current_task: tbd`** — The AGENTS.md task list exhausted. The last tasks were manually committed with `coach: "human"` status, bypassing coach review.
3. **No backlog generation** — Because the coach never ran, it never generated the next batch of tasks after the task list exhausted.
4. **Browser vision rate limited** — Even when the coach tried to run, `browser_vision` returned HTTP 429 (monthly usage limit), blocking visual comparison.

## What Was Actually Built
The action buttons WERE eventually added (commit `b972f0d`, Jun 17 17:46) via manual commits. The live site has them. The user's screenshot was from Jun 16 (before the fix).

## Fix Applied
- Reset checkpoint: all 45 completed tasks marked `coach: "approved"`
- Added 5 new tasks to AGENTS.md: preflop feedback flow, postflop GTO comparison overlay, keyboard shortcuts, random spot generator, progress tracking
- Set `current_task: study-preflop-action-feedback-flow`
- Pushed to main
- Triggered coach run manually

## Lessons for Coach Skill
1. **Stuck loop recovery** — Coach must check for `current_task: tbd` + all `coach: "approved"` as a STALLED state, not a silent state
2. **`coach: "human"` entries are unreviewed** — Coach must review them on next run
3. **Browser vision fallback** — When `browser_vision` 429s, use `browser_snapshot` as partial fallback and flag gap as "unverified"
4. **Coach cron reliability > model separation** — A coach that errors is worse than a coach that shares the player's model
