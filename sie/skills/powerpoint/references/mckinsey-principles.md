# McKinsey Presentation Principles
*Source: AI analysis of 1,000 real McKinsey slides*

## 8 Core Insights

### 1. Titles are Conclusions, Not Labels
- Average title length: **14 words**
- **72%** of titles are active and result-oriented — they tell the audience *why* the information matters
- **Anti-pattern:** "Q3 Revenue" → **Pattern:** "Q3 Revenue Exceeded Target by 12% Driven by Enterprise Segment"

### 2. Every Slide Has One Job
- **62%** of slides deliver a single, clear message
- Complex ideas are split across multiple slides rather than combined

### 3. Slides Should be Simple and Clear
- Average: **100 words** per slide
- Only **15%** were "dense" (200+ words) — mostly appendix slides

### 4. Use a Few Repeated Layouts
Four layouts = ~70% of all slides:

| Layout | % | Description |
|--------|---|-------------|
| Single Chart | 29.7% | Title + single chart |
| Chart + Bullets | 18.7% | Chart left, text right |
| Two-Column Comparison | 12.1% | Side-by-side |
| Table | 10.4% | Data table |

### 5. Use Visuals Extensively
- **71%** of slides contain a chart (~80% with frameworks)

### 6. Bar Charts are King
- Bar: **40%**, Line: **16%**, Waterfall: **11%**

### 7. One Chart is Usually Enough
- **59%** of chart slides use a single chart

### 8. Use Callouts on Charts
- **72%** of charts use callouts to highlight key takeaways

## Other Observations
- Minimal, intentional color
- Consistent templates (icons, colors, layouts)
- Top-down communication: conclusion in title, supporting details below

## Checklist
- [ ] Titles are conclusions, not labels
- [ ] One message per slide
- [ ] ~100 words per slide
- [ ] One of 4 core layouts
- [ ] Charts for data/concepts
- [ ] Bar charts for comparisons
- [ ] One chart per slide
- [ ] Callouts on charts
- [ ] Minimal intentional color
- [ ] Consistent template
