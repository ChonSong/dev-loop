# Session 2026-05-23 ŌĆö Roadmap Engine Zero-Output Root Cause

## What Happened

The Roadmap Engine ran Phase 1 ŌåÆ Phase 2 ŌåÆ Phase 3 without erroring, but produced **zero commits** and reported 0 tests passing. All 9 code tasks failed at the lint gate with the same error.

## Root Cause Chain

1. `roadmap.json` defines `proj-005` as:
   ```json
   {"id": "proj-005", "title": "openclaw monorepo", "repo": "ChonSong/agent-os", "local_path": "projects/openclaw"}
   ```
2. `roadmap_engine.py` line 37: `PROJECTS_DIR = WORKSPACE_TMP / "projects"` ŌåÆ resolves to `/tmp/hermes-sync/projects/`
3. All 9 `task-openclaw-*` tasks inherit `local_path` from `proj-005` ŌåÆ `/tmp/hermes-sync/projects/openclaw`
4. `_execute_code()` at line 577 runs `ruff check .` on `local_path` unconditionally before writing any code
5. The directory `/tmp/hermes-sync/projects/openclaw` does not exist ŌĆö it was never cloned
6. ruff exits code 1 with `[Errno 2] No such file or directory`
7. The lint gate interprets this as a lint failure, reverts any (non-existent) patch, and returns `status: "failed_qa"`
8. The error message is indistinguishable from a real lint error without inspecting the output prefix

**The real repo is `ChonSong/agent-os` (which exists at `/opt/data/hermes-sync/projects/agent-os`), but the clone URL used was `ChonSong/openclaw` (which does not exist on GitHub).**

## Why This Is Hard to Catch

- Phase 1 research doesn't test whether the repo directory exists on disk
- The lint gate is a blunt instrument ŌĆö it catches both "real lint errors" and "directory missing"
- The error output only shows the last 1500 chars of lint output, obscuring the `No such file or directory` prefix
- No step in the engine verifies `local_path` exists before attempting to execute code against it

## The Fix (Two Options)

**Option A ŌĆö Clone the correct repo into the right path:**
```bash
cd /tmp/hermes-sync && git clone https://github.com/ChonSong/agent-os.git projects/openclaw
```

**Option B ŌĆö Update roadmap.json to point proj-005 at the existing agent-os:**
Change `proj-005.local_path` from `projects/openclaw` to `projects/agent-os`, since the repo is already cloned at `/opt/data/hermes-sync/projects/agent-os`.

Option B is preferred ŌĆö it's a data-only fix, no clone needed.

## The Lint Gate False-Positive Pattern

When you see this in logs, it means `local_path` doesn't exist ŌĆö NOT a lint error:
```
[Code] LINT FAILED ŌĆö blocking commit. Output:
[Errno 2] No such file or directory: PosixPath('/tmp/hermes-sync/projects/openclaw')
```

The engine is working correctly; the problem is a missing preflight check.

## Cross-Mount Path Mismatch (Secondary Issue)

- `/opt/data/hermes-sync/projects/` has `repo-transmute` and `everything-dashboard`
- `/tmp/hermes-sync/projects/` only has `repo-transmute`
- `everything-dashboard` is mounted read-only at `/opt/data/hermes-sync/` and is NOT symlinked to `/tmp/hermes-sync/projects/`
- Phase 2 looking in `/tmp/hermes-sync/projects/` will report "Repo not found" for everything-dashboard even though it exists

This is the pre-existing read-only filesystem workaround creating a secondary shadow filesystem that gets out of sync.
