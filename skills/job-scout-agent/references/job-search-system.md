# Job Search System — Full Playbook

> A high-volume, structured approach to finding seated employment.
> User can oversee and manage the pipeline.

---

## Role Categories to Cycle Through

Assign each application a category. Aim for rough balance across the week.

### 1. PYTHON-BACKEND
- Roles: Junior/Associate Python Developer, Backend Engineer, API Developer
- Lead with: FastAPI, Python, REST/gRPC, Docker, PostgreSQL
- De-emphasise: Frontend, ML projects

### 2. FULL-STACK
- Roles: Junior Full-Stack Developer, Software Engineer, Web Developer
- Lead with: React, Vue.js, Next.js, FastAPI, TypeScript, Tailwind, Docker
- De-emphasise: ML/RL, data science theory

### 3. DATA / ENGINE
- Roles: Junior Data Engineer, Database Developer, Analytics Engineer
- Lead with: SQL Server, PostgreSQL, ETL pipelines, schema analysis, Streamlit
- De-emphasise: Frontend, Go projects

### 4. ML / AI
- Roles: Junior ML Engineer, AI Engineer, Graduate Data Scientist
- Lead with: RL algorithms (MCCFR, DPO), NumPy, Numba, model benchmarking
- De-emphasise: Frontend, general admin

### 5. GENERAL / GRADUATE
- Roles: Graduate Software Engineer, Tech Consulting Grad, Junior Developer
- Lead with: Broad range, degree, freelance delivery

## Daily Workflow (30 min)

1. **Open SEEK** (5 min) — paste 3 queries from different categories
2. **Scan & Pick** (5 min) — pick 5 roles that match, open in tabs
3. **Apply** (15 min) — paste matching CV variation, tweak 2 lines, submit
4. **Track** (5 min) — log all 5 in tracker CSV

## Weekly Review (Sunday, 15 min)

1. Count: 25+ applications this week?
2. Balance: hitting all categories?
3. Follow-up: anything from 7+ days ago?
4. Update statuses: interviews? rejections? ghosted?
5. Tweak: if a category gets zero responses, change approach

## Pitfalls

- Don't spray the same CV everywhere — categorise each application
- 3-5 apps/day minimum or volume doesn't work
- Follow up every application after 5-7 days
- Temp/contract roles are entry points, not fallbacks
- The "apply and forget" approach ghosted — track everything
