## Findings Discovered by Browser Testing

### Canvas Game Doesn't Always Render
The game serves via `<script type="module">` in the HTML, but the Hermes browser environment doesn't always execute module scripts on initial navigation. The page loads with `<head></head><body></body>` content — no canvas, no errors. A manual `import('/assets/index-<hash>.js')` is needed to trigger the game. This means real users in affected browsers see a blank page with zero feedback.

### City Menu Specific Bugs

1. **City menu persists through AI turn**: `startTurn()` nulls `selectedUnit` and `selectedHex` but never calls `hideCityMenu()`. The menu Group (depth 25) stays rendered throughout the AI's entire phase (multiple `renderAll()`/`updateUI()` cycles).

2. **Entity click priority — unit on city tile**: The click handler checks `findUnit(coord)` before `findCity(coord)`. If a unit stands on a city, clicking selects the unit and never reaches the city menu path. The city can only be accessed when no unit occupies it.

3. **Menu items scroll with camera (no setScrollFactor(0))**: City menu Text objects at line 1319 are positioned with `p.x + scrollX + 40` but created without `.setScrollFactor(0)`, so they're in world space. The position math doubles the scroll offset when the camera moves after creation.

4. **Playwright tests broken**: `tests/game.test.ts` imports `describe`/`it`/`expect` from vitest, but when `npx playwright test` runs, Playwright can't parse vitest globals. The E2E test suite has never run successfully. Additionally, `tests/gameplay.spec.ts` uses hardcoded pixel click positions `(466, 396)` that break on any map generation or camera offset change.

### Fix Recommendations

**For startTurn()**: Add `this.hideCityMenu()` at the top alongside `this.selectedUnit = null`.

**For click priority**: Check `findCity(coord)` before `findUnit(coord)` for the human player's own entities. If a city and friendly unit share a tile, show the city menu (the unit is implied to be inside).

**For menu scroll factor**: Either add `.setScrollFactor(0)` to all menu items and position at screen coords, or accept world-space positioning and destroy/recreate the menu after camera scrolls.

**For E2E tests**: Move vitest-only test files to `tests/unit/` and configure Playwright's `testMatch` to only look in `tests/e2e/`. Replace hardcoded pixel coordinates with hex-coordinate-to-screen-position utilities.
