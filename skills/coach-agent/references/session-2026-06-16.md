# Coach Session — Jun 16, 2026

## Discovery: Backlog Curation Pattern

The dev loop stalled because AGENTS.md tasks exhausted with no mechanism to generate the next batch. Key findings:

1. **Player-agent** had reactive Task Exhaustion Recovery but it was "don't touch AGENTS.md except when broken" — the player wouldn't act unless `current_task` was literally `"tbd"`.
2. **No one owned proactive backlog detection** — the loop would run until empty and then stop silently.
3. **Coach has freshest context** — it just reviewed what works and what doesn't. Extending coach to check remaining tasks at end-of-review is the minimal fix.

### Fix Applied

Three loaded skills patched:

- **coach-agent**: Added "Backlog Health Check" section after the review verdict. Coach checks remaining tasks, and if < 2, investigates what's broken (parallel-investigation pattern), generates 3-5 new tasks, appends to AGENTS.md, updates checkpoints, commits, pushes.
- **player-agent**: Added "Proactive Backlog Check" at start of tick. Checks remaining unstarted tasks. 1 remaining = warn, 0 remaining = trigger Task Exhaustion Recovery (safety valve if Coach missed generating).
- **doc-driven-dev-loop**: Added "Backlog Curation" section (who owns what) and "Skills-First Approach" principle (research existing skills before proposing solutions).

### Skills Composed for Backlog Generation

| Skill | Purpose |
|-------|---------|
