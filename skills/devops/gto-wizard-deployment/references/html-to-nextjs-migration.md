# Generated HTML → Next.js Page Conversion

## When to Use This Pattern
A user provides a standalone HTML file (generated by a frontend model) and wants it integrated into the existing Next.js app. The HTML has all CSS, markup, and JS inline.

## Workflow

### 1. Read and Map
- **CSS variables**: Extract the `:root` block — color palette, spacing, fonts. Most models generate consistent `#0b0d0f`/`#00a98f` design.
- **Layout structure**: Identify the top-level containers (`.app`, `.topbar`, `.panel`, `.grid`)
- **State variables**: Find all `let` declarations and the primary state object
- **Event handlers**: Map `onclick=` and `addEventListener` calls to React handlers
- **DOM queries**: Note every `getElementById`, `querySelector`, and `innerHTML` assignment
- **Game engines**: Identify any standalone computation (Monte Carlo, evaluators, shufflers)

### 2. Build the Next.js Page
Create a single `'use client'` page component at `apps/web/src/app/{route}/page.tsx`:
- Replace all CSS with inline `style={{}}` objects matching the reference CSS variables
- Convert `document.getElementById('foo')` → `useState`/`useRef`
- Convert `.innerHTML = ...` → JSX elements
- Convert `onclick` → `onClick` React handlers
- Keep game engines as plain TypeScript functions (no JSX needed)
- Add mock data fallback for API calls

### 3. Avoid These Pitfalls
- **Tailwind imports** — The generated HTML uses raw CSS, NOT Tailwind. Don't add `@/lib/utils` or `cn()` imports. Use inline styles directly.
- **Chart CDNs** — Generated pages often use Chart.js from CDN. Replace with simple Canvas drawing or omit charts on first pass.
- **LocalStorage** — Pages often use `localStorage.getItem('gtoPlaySession')`. Convert to `useState(() => { try { return JSON.parse(localStorage.getItem('key')) } catch { return defaultValue } })` in a useEffect for writes.
- **Font imports** — The app loads Inter from Google Fonts in layout.tsx. Don't duplicate in individual pages.

### 4. Design System Reference
When the generated HTML uses slightly different colors than the app's established palette:
- App default: `--bg: #0b0d0f`, `--panel: #14171b`, `--accent: #00a98f`
- Generated HTML may use: `--bg: #0A0A0A`, `--panel: #1C1C1C`, `--accent: #22c55e`
- **Prefer the generated HTML's color scheme** — the model designed it for a reason. Slight inconsistencies between pages are fine; each page stands alone.

### 5. Verification
```bash
# Check the page loads
curl -s -o /dev/null -w "%{http_code}" http://localhost:8564/{route}

# Check the title
curl -sL http://localhost:8564/{route} | grep -o '<title>[^<]*</title>'

# Check key content renders
curl -sL http://localhost:8564/{route} | grep -oP 'Header|KeyComponent|Button Label'
```
