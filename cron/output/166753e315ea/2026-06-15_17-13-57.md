### 🔴 Issue Found & Fixed

**Problem:** `GET /api/v1/courses` (and all `/courses/*` endpoints) returned **500 Internal Server Error**.

**Root cause:** The backend falls back to SQLite (`gto_wizard.db`) when no `DATABASE_URL` env var is set. The database connection uses `sqlite+aiosqlite:///` but the **`aiosqlite` Python package was not installed** in the virtual environment. SQLAlchemy's lazy engine initialization meant requests that hit the database failed with a missing-module error.

**Fix:** Installed `aiosqlite==0.22.1` via `uv pip install aiosqlite`. All course endpoints started working immediately without a backend restart (the import is lazy).
